# 关于 JEP

- jep 0: jeps 的目录。
- jep 1: 定义了一个流程, 用于收集, 审查, 排序和记录 jdk 改进建议的结果, 以及相关工作(如流程和基础设施改进)的结果。
- jep 2: jep 模版。
- jep 3: 定义 OpenJDK 社区中贡献者生成基于时间, 快捷做的 jdk 特性版本的过程。
- jep 11: 孵化器模块, 将非最终 API 交付给开发者的方法, 在未来的 API 中, 要么完成, 要么删除。
- jep 189: 添加 `Shenandoah` 垃圾收集算法, 通过与正在运行的 Java 线程并发执行回收工作来减少 GC 暂停时间。 `Shenandoah` 暂停时间与堆大小无关, means 200MB 与 200GB 的 heap, 暂停时间相同。
- jep 230: 在 jdk 源码中添加一套基本的 微基准测试, 方便的直接使用或者创建新的 微基准测试。
- jep 325: 关于 Switch Expressions, 扩展 switch 语句, 使其既可以用作语句, 也可以用作表达式。 两种行为都可以使用 "传统的" 或 "简化的" 作用与和控制流行为。
- jep 326: 关于字符串 preview。 向 Java 中添加原始字符串文本, 原始字符串可以跨越源代码的多行, 且不解释转移序列。 预览语言的功能。
- jep 334: 引入一个 API 来建模关键类文件和运行时 artificts 的描述, 特别是可以从常量池中加载的常量。
- jpe 340: 移除对 arm64 端口相关的所有的 source, 保留 32 位 arm 端口和 64 位 aarch64 端口。
- jep 341: 增强 jdk 构建过程, 在 64 位平台上使用默认的类列表生成类数据共享存档(CDS)。
- jep 344: 当 G1 垃圾回收器的回收超过暂停目标, 则能中止垃圾回收过程。
- jep 346: 增强 G1 垃圾收集器, 以便在空闲时自动将 Java Heap 内存返回到操作系统。
